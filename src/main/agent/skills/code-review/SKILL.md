# Code Review Skill

## When to Use
当用户要求：
- 审查代码
- 检查代码质量
- 寻找 bug 或问题
- 代码优化建议

## How to Use
1. 用户提供代码文件路径或直接粘贴代码
2. 分析代码结构、命名、逻辑
3. 指出潜在问题（bug、安全漏洞、性能问题）
4. 提供具体的改进建议

## Output Format
以结构化方式输出：
- **概览**: 代码整体评估
- **问题**: 发现的问题列表
- **建议**: 具体改进方案
- **评分**: 1-10 分的代码质量评分

## Example
用户: "帮我审查 src/utils/api.ts"
输出: 按上述格式分析该文件
